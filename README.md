## 🚀 Native Architecture Features (EXPERIMENTAL)

This fork introduces a **native Kubernetes execution architecture** that replaces Docker-in-Docker with:

- **Native Container Execution**: Run network devices as standard Kubernetes Deployments
- **KubeVirt VM Support**: Execute VM-based appliances using KubeVirt VirtualMachines  
- **Cilium CNI Integration**: Advanced networking with eBPF dataplane and NetworkPolicies
- **Intelligent Workload Detection**: Automatic classification of container vs VM workloads
- **Mixed Topologies**: Support for hybrid topologies with both containers and VMs
- **Per-Node Overrides**: Fine-grained control over execution modes and resources

**Key Components:**
- `pkg/executor/`: Core execution interfaces for containers and VMs
- `pkg/networking/cilium/`: Cilium integration and NetworkPolicy generation
- `pkg/workload/`: Workload classification, rendering, and reconciliation
- Extended APIs with `NativeExecution` configuration

**⚠️ Remember: This code is experimental and largely LLM-generated. Use with caution!**
